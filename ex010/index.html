Making CSS Grid Responsive

Instead of relying only on media queries, you can use auto-fit or auto-fill with the repeat() function in CSS Grid. This makes your grid responsive and cleaner, because the number of columns adjusts automatically based on the available space. However, for more complex templates or pages, we can combine this technique with media queries. For example, if you want an element to span multiple columns or rows at a certain browser width, you’ll need media queries to handle those specific changes. EX of the function would be grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

auto-fill → Fills the row with as many columns as possible, even if some are empty. This means it will keep the grid tracks reserved, even if there are no items to fill them. If the container has extra space but fewer items than possible columns it keeps the empty columns (like invisible boxes).

auto-fit → Similar to auto-fill, but it collapses empty columns so the content stretches to take up the extra space. If the container has extra space but fewer items than possible columns IT collapses them and expands the existing items.

When do we still need media queries?

When Making an element span multiple columns or rows only at larger screen sizes or when changing the layout significantly at different breakpoints.
