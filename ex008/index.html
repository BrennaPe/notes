Meta tags

Meta tags provide information about your HTML document. This information (called metadata) isn’t shown directly on the page, but it helps browsers, search engines, and devices understand how to handle your content.
You should place all meta tags inside the <head> section of your HTML.

What can Meta Tags do?

Why use Meta Descriptions?

A meta description is a short summary of a webpage. It appers under the page title in search engine results
Benefits:

  1. More Clicks - A clear and attractive summary can encourage users to visit your page.
  2. Clarity - It helps users quickly understand what the page is about
  3. Better results - Search engines may use your meta description as a preview.

EXAMPLES

Set the character encoding (e.g., UTF-8) So the browser can display accents, symbols, and emojis correctly.
<meta charset="UTF-8">

Tell search engines what your page is about (keywords & description). It Helps with SEO (Search Engine Optimization).
<meta name="description" content="Learn what HTML meta tags are and how to use them. Great for SEO and accessibility.">
<meta name="keywords" content="HTML, meta tags, SEO, web development, coding, tutorial">

Say who the author of the page is. It tells who wrote or owns the content.
<meta name="author" content="Your Name">

Automatically refresh the page after a few seconds. It is good for live dashboards or temporary pages.
<meta http-equiv="refresh" content="10">. This will refresh the page every 10 seconds. You can change the number.

Make your website responsive on phones and tablets (viewport tag). Ensures your site adapts to smaller screens.
< meta name="viewport" content="width=device-width, initial-scale=1.0">

Prevent your site from being automatically translated by browsers. Stops Google Translate or other tools from auto-translating.
<meta name="google" content="notranslate">

Control how browsers cache your pages. To control whether the browser stores your page for speed or always loads the latest version.
<meta http-equiv="Cache-Control" content="no-cache">
<meta http-equiv="Pragma" content="no-cache">
<meta http-equiv="Expires" content="0">

What is Browser cache?

When you visit a website, yout browser (like Chrome or Safari) stores so me files on your computes. This includes things like: Images, Stylesheets (CSS), JavaScript files and even some HTML. This is called caching.

Why does the crowser cache files?

Intesd of downloading everything again from the internet, the browser just uses the local (already saved) files. So, the bbrowser cache files to make the site load faster the next time you visit.

Caching become a problem when: You update your site, the visitors still see the old version. You want people to always see the the newest content.

To prevent browsers from using cache

To prevent browsers from using cache (and always load the newest version of your site), you can add these three meta tags inside your >head< section:
<meta http-equiv="Cache-Control" content="no-cache">
<meta http-equiv="Pragma" content="no-cache">
<meta http-equiv="Expires" content="0">
These meta tags tell the browser: “Don’t save anything. Always get the fresh version.”

What do they mean?

Meta Tag What it Does
Cache-Control: no-cache Asks the browser to revalidate the content with the server before using cache.
Pragma: no-cache An older version of Cache-Control. Still used for compatibility.
Expires: 0 Says the content is already expired, so the browser must reload it.

What It does?

  • Cache-Control: no-cache: Browser must check with the server before using anything from the cache.
  • Pragma: no-cache: Backup for older browsers.
  • Expires: 0: Tells browser: “This page is already expired, reload it.”

Use these especially when your content changes often or you’re testing updates during development.

Example to allow caching for 1 hour

<meta http-equiv="Cache-Control" content="no-cache">
<meta http-equiv="Cache-Control" content="public, max-age=3600">
<meta http-equiv="Expires" content="Tue, 01 Jan 2030 00:00:00 GMT">
<title>Cache Enabled Example</title>

What this does:

  • Cache-Control: public, max-age=3600
  • llows browsers and proxies to cache the content for 3600 seconds (1 hour).
  • Expires: [future date]. Sets an exact date/time when the content should be considered “expired” and rechecked.
