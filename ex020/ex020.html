Tipos Primitivos JS

O que são tipos primitivos?

Eles são chamados de primitivos porque armazenam valores imutáveis e não possuem métodos próprios (exceto symbol). Os tipos primitivos no JavaScript são os tipos mais básicos de dados que podemos usar. Eles são chamados de primitivos porque:

  1. São Simples: Não têm estrutura complexa, como objetos ou arrays
  2. São imutáveis: Depois que um valor primitivo é criado, ele não pode ser alterado diretamente.
  3. Não têm métodos próprios: Exceto o tipo Symbol, que tem alguns métodos especiais. Tipos primitivos não têm métodos próprios, mas o JavaScript “empresta” métodos temporários para usarmos.

Pense assim:Um primitivo é como uma fotocópia, você pode usar e criar novas, mas não pode mudar a original.

Um exemplo prático seria var nome = "Maria"; nome.toUpperCase(); Resultado= MARIA // Este comando transforma a variavel desejada em maiúsculas. Perceba que ainda será "Maria"! O valor original NÃO mudou pois string é um tipo primitivo..

7 tipos primitivos de dados.

String (string)

Representa textos e deve estar entre aspas simples ('), duplas (") ou crase (` para template literals).

Tamplate literals são strings (textos) que usam crases (`) no lugar de aspas simples (') ou duplas (").

Com Template Literals, você pode: Quebrar linhas dentro da string sem precisar de \n, inserir variáveis direto no texto usando ${ } (sem precisar de +) e criar strings dinâmicas de forma mais fácil e organizada.

Tamplate Literals

Exemplo Normal (sem Template Literals) Usando concatenação (com +):

var nome = "Ana";
var idade = 25;
var mensagem = "Olá, meu nome é " + nome + " e eu tenho " + idade + " anos.";
console.log(mensagem);
Olá meu nome é Ana e e tenho 25 anos.

Perceba que muitas aspas e + tornam o código difícil de ler.

Exemplo com Template Literals Usando crases e ${}:

var nome = "Ana";
var idade = 25;
var mensagem = `Olá, meu nome é ${nome} e eu tenho ${idade} anos.`;
console.log(mensagem);
Olá meu nome é Ana e e tenho 25 anos.

Nota-se que sem precisar concatenar (+), o código fica mais limpo e legíve

Outro exemplo: Quebrando linhas facilmente

Sem Template Literals:
let texto = "Linha 1\nLinha 2\nLinha 3";
console.log(texto);

Com Template Literals:
let texto = `Linha 1
Linha 2
Linha 3`;
console.log(texto);

Undefined (undefined)

Quando uma variável existe, mas não tem valor atribuído.

Null (null)

Representa ausência de valor (intencional)

A diferença entre null e undefined é que o undefined a variável foi declarada, mas não tem valor atribuído. Já o null o valor atribuído é intencionalmente “vazio”.

Atribuição de valor a uma variável

O que é “atribuir um valor” a uma variável?

Quando criamos (declaramos) uma variável, ela existe, mas ainda não tem um valor até que a gente defina um.

Exemplo sem valor atribuído (undefined):
var nome; // A variável "nome" foi criada, mas ainda não recebeu um valor
console.log(nome); // undefined
Aqui, nome foi declarada, mas como não demos um valor a ela, o JavaScript coloca undefined automaticamente.

Exemplo com valor atribuído:
let nome = "Ana"; // Agora a variável tem um valor atribuído
console.log(nome); // "Ana"
Agora, nome tem um valor (“Ana”), então não é mais undefined.

Symbol (symbol)

Usado para criar valores únicos que não podem ser alterados.

Entendendo Symbol

O que é Symbol?

O Symbol é um tipo de dado especial que cria identificadores únicos. Imagine que você precisa dar um rótulo (ou etiqueta) para algo no seu código, mas quer ter certeza absoluta de que ninguém mais usará esse mesmo rótulo.

Exemplo
let id1 = Symbol("id"); // Cria um símbolo
let id2 = Symbol("id"); // Outro símbolo, mesmo nome, mas diferente!
console.log(id1 === id2); // false (são diferentes!)
Mesmo que tenham o mesmo nome ("id"), são símbolos diferentes e únicos.

BigInt (bigint)

Usado para representar números inteiros muito grandes, além do limite do Number. O BigInt é usado para armazenar números muito grandes, que não cabem no tipo Number.

No JavaScript, o maior número seguro que podemos representar com Number é 2⁵³ - 1 (aproximadamente 9 quintilhões!). Se precisar de algo maior, usamos BigInt.
